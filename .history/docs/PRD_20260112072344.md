- Supports both virtual and in-person play
- Offers a growing library of party games
- Remains free forever through sustainable, low-cost architecture

## ðŸš« Non-Goals (What We're NOT Building)

### Explicitly Out of Scope

1. **No User Accounts**
   - No login, no passwords, no profiles
   - One-time sessions only
   - Reasoning: Simplicity, privacy, faster start

2. **No Monetization**
   - No ads
   - No premium features
   - No in-app purchases
   - Reasoning: Commitment to accessibility

3. **No Large-Scale Lobbies**
   - Max 16 players per game
   - Not building for 50+ player events
   - Reasoning: Party game focus, not massive multiplayer

4. **No Complex 3D Graphics**
   - 2D card games and simple visuals
   - Not building immersive 3D worlds
   - Reasoning: Performance, accessibility, scope

5. **No Persistent Progression**
   - No levels, no unlocks, no achievements (initially)
   - Each game is standalone
   - Reasoning: Casual focus, no account system

6. **No Mobile Game Mechanics**
   - Not building Candy Crush-style games
   - Not building idle games or gachas
   - Reasoning: Party game focus

7. **No Server Dependency (Initially)**
   - Games work offline when possible
   - P2P or client-side logic preferred
   - Reasoning: Cost, reliability, accessibility

## ðŸ”„ User Flows

### Critical User Flows

#### Flow 1: New User - First Game (Single Player)
```
1. User visits game-night.app
2. Sees landing page with 3 tabs
3. "Single Player" tab is default
4. Clicks "UNO: No Mercy" card
5. Setup screen:
   - Enter name (default: "Player 1")
   - Select difficulty (default: Medium)
   - Select # of opponents (default: 2)
6. Click "START GAME"
7. Game loads <2 seconds
8. Tutorial tip (first time only): "Play cards matching color or number"
9. Start playing!

Success Criteria:
- <30 seconds from landing to playing
- Zero confusion or help needed
- Settings remembered for next time
```

#### Flow 2: Friend Group - Imposter Game (In-Person)
```
1. Host visits app on phone
2. Switches to "In-Person" tab
3. Clicks "Imposter" card
4. Setup screen:
   - Add player names (drag to reorder)
   - Select # of imposters (default: 1)
   - Select category (default: All)
5. Click "START GAME"
6. Each player views their role on host device
7. Word/role reveals happen
8. Group plays, discusses
9. Voting screen
10. Results reveal
11. "Play Again" â†’ back to step 5

Success Criteria:
- Setup <2 minutes
- No technical questions from players
- Host can control everything
- Clear role reveals (no peeking)
```

#### Flow 3: Remote Friends - Virtual Game (Future)
```
1. Host creates party
2. Gets shareable link/code
3. Friends join via link
4. Host selects game
5. Everyone sees game on their device
6. Real-time sync
7. Play together
8. Return to lobby for next game

Success Criteria:
- <1 minute to get everyone in
- Zero sync issues
- Low latency (<100ms)
```

### Key Technical Decisions

**Frontend:**
- **Framework:** React 18+
- **Router:** React Router v6
- **Animation:** Framer Motion
- **Styling:** CSS Modules + Theme Context
- **State:** React Hooks (useState, useEffect)
- **Build:** Create React App / Vite

**Game Logic:**
- **Pattern:** Separate logic from UI (GameState classes)
- **Location:** `/src/logic/[game-name]/`
- **Reasoning:** Reusable across platforms, testable

**Storage:**
- **Method:** localStorage (browser)
- **Wrapper:** StorageManager utility
- **Scope:** Settings, player names, preferences
- **Limit:** 5-10MB (browser dependent)

**Hosting:**
- **Platform:** Vercel (free tier)
- **CDN:** Automatic (Vercel Edge Network)
- **Domain:** Custom domain
- **SSL:** Automatic HTTPS

**Future (Virtual Mode):**
- **Backend:** Node.js + WebSocket (Socket.io)
- **Hosting:** Heroku/Railway/Fly.io free tiers
- **Fallback:** P2P WebRTC (no server)
