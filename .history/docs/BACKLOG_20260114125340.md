## üìù Notes & Conventions

### Priority Definitions

- **P0 (üö®):** Critical - blocks core functionality, fix immediately
- **P1 (üî•):** High - important bugs or features, fix this sprint (1-2 weeks)
- **P2 (‚≠ê):** Medium - planned features, do this month
- **P3 (üí°):** Low - nice-to-haves, backlog for later

### Labels

- **[BUG-XXX]:** Something broken
- **[FEAT-XXX]:** New feature
- **[DESIGN-XXX]:** Visual design work
- **[UX-XXX]:** User experience improvement
- **[GAME-XXX]:** New game implementation
- **[SOCIAL-XXX]:** Multiplayer/social features
- **[PLATFORM-XXX]:** Platform-specific (iOS, Android)
- **[POLISH-XXX]:** Quality of life improvements
- **[ARCH-XXX]:** Architecture/infrastructure
- **[UI-XXX]:** UI component work

### Update Guidelines

1. **After fixing a bug:** Check the box, move to "Recently Completed"
2. **After shipping a feature:** Check the box, move to "Recently Completed"
3. **When discovering a bug:** Add to appropriate priority section
4. **When getting a feature idea:** Add to P3 with effort estimate
5. **Weekly:** Review priorities, move urgent items up
6. **Monthly:** Archive old completed items
